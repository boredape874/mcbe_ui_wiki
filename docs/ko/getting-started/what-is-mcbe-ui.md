# MCBE UI란?

::: info 이 페이지에서 배울 것
- 마인크래프트 베드락 에디션에서 "리소스"란 무엇인가
- 리소스팩이란 무엇인가
- 리소스팩 안의 UI 파트가 무엇인가
:::

## 리소스(Resource)란?

마인크래프트 베드락 에디션(MCBE)에서 **리소스**란 게임이 화면에 무언가를 그리거나 소리를 낼 때 사용하는 모든 파일을 말합니다.

| 종류 | 설명 | 예시 |
|------|------|------|
| 텍스처 | 블록, 아이템, 몹 등의 이미지 | `textures/blocks/stone.png` |
| 사운드 | 효과음, 음악 | `sounds/mob/zombie/say1.ogg` |
| 모델 | 블록과 아이템의 3D 형태 | `models/blocks/stone.geo.json` |
| UI | 인게임 화면 구성 | `ui/hud_screen.json` |
| 폰트 | 게임 내 글꼴 | `font/glyph_default.png` |

기본 게임에는 이미 바닐라 리소스가 내장되어 있어, 이 파일들이 있어야 게임이 정상적으로 보이고 들립니다.

## 리소스팩(Resource Pack)이란?

**리소스팩**은 게임의 기본 리소스를 교체하거나 추가하는 파일 묶음입니다.

리소스팩을 적용하면 게임 파일 자체를 건드리지 않고도 게임의 외형을 바꿀 수 있습니다. 리소스팩은 바닐라 리소스 위에 **덮어쓰는** 방식으로 동작합니다.

```
게임이 stone.png를 불러올 때:
  1. 적용된 리소스팩에 stone.png가 있는가?
     → 있으면: 리소스팩의 파일 사용
     → 없으면: 바닐라 기본 파일 사용
```

리소스팩의 기본 폴더 구조는 다음과 같습니다.

```
my_resource_pack/
├── manifest.json        ← 팩 정보 (이름, 버전, UUID 등)
├── pack_icon.png        ← 팩 아이콘
├── textures/            ← 텍스처 파일
├── sounds/              ← 사운드 파일
├── models/              ← 모델 파일
├── ui/                  ← UI 파일
└── font/                ← 폰트 파일
```

## 리소스팩의 UI 파트

리소스팩 안의 `ui/` 폴더는 **인게임 화면을 정의하는 JSON 파일**들로 구성됩니다.

```
ui/
├── _ui_defs.json           ← 사용할 UI 파일 목록 등록
├── _global_variables.json  ← 전역 변수 정의
├── hud_screen.json         ← 인게임 HUD 화면
├── start_screen.json       ← 타이틀 화면
├── inventory_screen.json   ← 인벤토리 화면
└── ...
```

이 JSON 파일들이 바로 **JSON UI**입니다. JSON UI를 수정하면 체력바, 핫바, 채팅창, 메뉴 화면 등 게임의 모든 UI를 원하는 대로 바꿀 수 있습니다.

::: tip 부분 오버라이드
리소스팩의 UI 파일은 바닐라 파일을 통째로 교체하지 않아도 됩니다. 수정하려는 요소만 골라서 덮어쓰는 **부분 오버라이드** 방식도 지원하므로, 필요한 부분만 작성하면 됩니다.
:::

## 정리

```
마인크래프트 베드락
└── 리소스 (Resource)
    └── 리소스팩 (Resource Pack)
        └── ui/ 폴더
            └── JSON UI  ←  이 위키에서 다루는 주제
```
